# IDM 第一阶段验证总结

## 目标

验证逆动力学模型（IDM）能否从前后两帧截图推断中间发生的动作类型和坐标。

## 实验设置

- **数据**：Windows 文件管理器，5 种操作类型（click / dblclick / drag / key / scroll），共 1552 个样本
- **模型**：双流轻量 CNN（SimpleEncoder），分类头 + 坐标回归头
- **输入分辨率**：224×224 → 最终提升至 672×672
- **帧策略**：从 t±1/fps 调整为 t-3/fps 和 t+3/fps

## 结果

### 分类

表面上训练集和验证集准确率均达到 100%，但新录制数据上只有 click 类型接近 100%，其余类型接近 0%。

**真实结论：模型没有学会任何动作类型。** 模型将所有输入预测为 click，因为训练集中 click 比例最高，这是最省力的策略，不是真正的泛化。

### 坐标回归

训练集平均误差约 300px，新数据上约 350-450px，远超 20px 的目标。

提高分辨率和调整帧策略均无明显改善。根本原因是 CNN 的多次下采样导致特征图空间分辨率严重损失（672×672 → 84×84），无法精确定位操作坐标。

## 根本原因

轻量 CNN 从零训练，缺乏预训练的视觉语义理解能力，1552 个样本远不足以学到跨录制的通用视觉特征。

## 结论

第一阶段验证**未通过**，但失败原因明确：

- 编码器太弱，需要预训练视觉编码器
- 坐标回归需要保留空间信息的架构
- IDM 思路本身没有问题，数据飞轮逻辑成立

## 下一步

用 **SigLIP** 替换 SimpleEncoder，重新验证 IDM。SigLIP 具备开箱即用的 GUI 视觉理解能力，少量数据即可泛化，是文档中原本设计的编码器方案。
